Um eine SQL Injection durchzuführen muss als erstes
verstanden werden, wie die SQL Syntax aussehen könnte
bzw. wie ein Server die Anfrage bearbeitet.
Bei unsicheren Systemen wird häufig eine Wert direkt
an die Datenbank geschickt statt einen eingegebnen Wert zu
vergleichen. Das führt dazu, dass die Datenbank auch SQL
Syntax ausführen kann, weil der Wert ohne hindernisse in
eine SQL Abfrage eingefügt wird. Um sich das besser vorzustellen
kann man als Beispiel das Paswort feld als Wert:
Passwworteingabe = Deine Eingabe (z.B 123456) sehen.
Bei dem unsicheren system wird der wert so an die Datenbang geschickt:
SELECT * FROM datenbank WHERE Passworteingabe='123456';
Der Wert ist in Anführungszeichen, damit es als String (Zeichenkette) interpretiert wird.
