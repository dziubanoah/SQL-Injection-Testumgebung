Was ist SQL ?

- SQL ermöglicht das Abfragen von Daten in Datenbanken.

- Wird zur Überprüfung von eingegebenen Daten genutzt.

- Vergleicht eingegebene und gespeicherte Daten.

- Ermöglicht Zugriff bei korrekten Daten.

Was ist eine SQL Injection?

- Tritt bei unsicherem Backendsystemen auf.

- SQL-Anfragen werden direkt weitergegeben.

- Angreifer fügen SQL Befehle ein.

- Ermöglicht unbefugten Zugriff.

OR Statement in SQL

- SQL kann Alternativen für Eingabewerte nutzen.

- OR fügt eine zusätzliche Bedingung hinzu.

- Beispiel: Passwort OR 1=1 gibt eine gültige Eingabe.

- Kann unbefugten Zugriff ermöglichen.

Wie interpretiert SQL ein Passwort ?

- Passwörter werden unverschlüsselt gespeichert.

- SQL-Syntax darf nicht im String enthalten sein.

- Umgehung vom String durch das Zeichen '.

- Beispiel: Pw' OR 1=1 wird als SQL-Befehl erkannt.

SQL Code auskommentieren

- Code hinter `--` wird ignoriert.

- So wird alles hinter OR 1=1 unsichtbar.

- Verhindert die Ausführung weiteren Codes.

- Beispiel: Pw' OR 1=1 -- macht weiteren Code unsichtbar.

Weitere Infos

- Jede SQL-Abfrage benötigt ein Semikolon zum Abschließen.

- Beim Auskommentieren wird das Semikolon entfernt.

- Es muss manuell hinzugefügt werden: Pw' OR 1=1; --

- Beispiel: 'Pw' OR 1=1; -- auskommentierte Anweisung;.

- Wichtig: Vor und nach `--` muss jeweils ein Leerzeichen.
